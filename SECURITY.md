## Security Policy

Please report security vulnerabilities privately. Do not open a public issue for sensitive vulnerabilities.

## Supported Versions

Security fixes are prioritized for the latest code on `main`.

## How To Report A Vulnerability

* Open a private GitHub security advisory for this repository.
* Include impact, reproduction steps, and suggested mitigation if known.
* Provide logs, screenshots, or proof of concept only if they do not expose secrets.

## What To Expect

* Initial acknowledgement within 3 business days
* Triage and severity assessment after acknowledgement
* Coordinated fix and disclosure timeline based on impact

## Scope

In scope:

* Application code in this repository
* Build and deployment configuration in this repository

Out of scope:

* Social engineering attacks
* Vulnerabilities in third-party services outside repository control
